by Lucy Block

This week, I used the command line to pull Housing New York data on buildings and projects from the NYC Open Data portal API. I added those CSVs to Carto and used the built-in Analysis feature to join those datasets to the DHCR data from last week. I used an intersect/inner join to select only properties that appear in both DHCR's data and Housing New York, because I wanted to examine which programs they're listed in in each to look at consistencies. A visual check in the map and doing some simple SQL queries on the dataset within Carto suggest that the two datasets are almost completely inconsistent in which programs they display. E.g. the properties that show an Article XI tax benefit in the HNY dataset mostly show J-51 in the DHCR dataset. Many properties in the HNY dataset show 420c, which isn't in the DHCR data at all. I need to look into what that program is.

My issues arose mostly from the Carto platform, which I always struggle with. I find it so easy to break things using their point-and-click interface. For example, I used their interface to join layers and then deleted what I thought was an extraneous layer from the map, but because I had pointed to the layer within the map rather for the join rather than the dataset outside the map, I broke and deleted my entire analysis. I don't find it intuitive and they don't give you warnings for that type of thing.


Data Viz critique

I wanted to focus on this I Quant NY map not because it’s an incredible visualization, but because it comes from a data source that I already use for work (e.g. in DAP Map) and am interested in exploring further in this class. It’s a dataset of rent stabilized unit counts scraped from DOF tax bill PDFs. The blog post is more of an initial data exploration than a complete analysis in my opinion.

I think the simplicity is helpful as a quick way to break down the data for a public audience. However, Ben Wellington mostly takes the data at face value. I’ve compared John Krauss's tax bills data to Rent Guidelines Board reports and found huge discrepancies when aggregating the data by year. So he’s presenting increases and decreases in different geographies, but based on that other data source I’m not convinced there aren’t major biases skewing those conclusions. What he did makes absolute sense if you’re analyzing the data source by itself, I just know that this particular one might have caveats that aren’t addressed and can only be figured out by digging into other sources.
